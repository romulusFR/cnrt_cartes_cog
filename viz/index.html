Visualisation du thésaurus avec Vega

On représente graphiquement les quatres niveaux (mots énoncés - code 4, mots concepts - code 3, mots mères - code 2, mots grands-mères - code 1) du thésaurus (source) avec https://vega.github.io/vega/ à partir des données calculées en Python. Pour déterminer les poids des mots, on utilise ici les quatres cartes obtenues à partir des mots-énoncés de la mine (source) avec le système de poids inverse de la forme \(1/n\) où \(n\) est la position dans la carte (source).

La liste des visualisations disponibles (passer la souris sur les visualisations pour avoir les info-bulles avec quelques détails) :
  • vue sunburst
  • vue tree
  • vue radial
  • vue treemap
  • vue circle-packing
  • vue wordcloud
  • vue histogramme
  • vue heatmap dynamique

Intuitivement, avec l'échelle log-log, on a une relation \(\log(c) = a*\log(w) + b\) avec \(a < 0 \), où \(c\) est le nombre de mots et \(w\) le nombre d'occurrence, ce qui indiquerait une Zipf's law. Ici c'est pour un système de poids inverse, est-ce que la loi résiste avec d'autres systèmes de positions pondérées ? \begin{align} c &= \exp(\log(c))\\ &= \exp(a*\log(w) + b)\\ &= \exp(b)*\exp(a*\log(w))\\ &= \exp(b)*\exp(\log(w))^a\\ &= \exp(b)*w^a \end{align}
